# 06 — iOS Widget Extension

iOS gets a different experience from Android. Apple's sandbox forbids drawing over other apps and listening for unlock events, so instead the app ships a **WidgetKit home-screen widget** that the user adds manually.

## File layout

```
ios/
└── App/
    ├── App/
    │   ├── AppDelegate.swift             ← stock Capacitor delegate, no customization
    │   ├── Info.plist                    ← bundle display name "ProphetSalah"
    │   └── Assets.xcassets
    ├── App.xcodeproj                     ← Xcode project (Capacitor-generated)
    ├── CapApp-SPM/                       ← Capacitor SPM scaffolding
    └── WidgetExtension/
        └── WidgetExtension.swift         ← The entire widget (~500 lines, single file)
```

## Manual setup (NOT done by `cap sync`)

The widget target must be added by hand because Capacitor doesn't manage it, and edits to `.pbxproj` via tooling can corrupt the project. From `iOS_SETUP_GUIDE.md`:

1. Open `ios/App/App.xcworkspace` in Xcode (`npx cap open ios`).
2. **File → New → Target → Widget Extension**.
3. Name: `WidgetExtension`. **Uncheck** "Include Live Activity" and "Include Configuration App Intent".
4. Delete the autogenerated Swift files in the new target group.
5. Right-click the WidgetExtension folder in Xcode → **Add Files to "WidgetExtension"** → pick `ios/App/WidgetExtension/WidgetExtension.swift`. Ensure target is `WidgetExtension` only.
6. **Enable App Groups** on both the `App` target and the `WidgetExtension` target:
   - Signing & Capabilities → + Capability → App Groups
   - Add `group.com.salo.alahmoha` to both
7. Build & run.

If you change the App Group ID, update **all three places**: both targets in Xcode and the `suiteName` literal in `WidgetExtension.swift:374`.

## How the widget works

### Entry point

```swift
@main
struct ProphetSalahWidget: Widget {
    let kind: String = "ProphetSalahWidget"

    var body: some WidgetConfiguration {
        StaticConfiguration(kind: kind, provider: Provider()) { entry in
            WidgetExtensionEntryView(entry: entry)
        }
        .configurationDisplayName("Salah Reminder")
        .description("A premium reminder to send blessings on the Prophet.")
        .supportedFamilies([.systemSmall, .systemMedium])
    }
}
```

Static config (no user-configurable parameters), small + medium families only (no large/lockscreen).

### TimelineProvider

`Provider.getTimeline(in:context:completion:)` is the heart of the widget:

```swift
1. Read user_lang from UserDefaults(suiteName: "group.com.salo.alahmoha")
   → fallback "ar"
2. pool = isArabic ? SalahPhrases.arabic : SalahPhrases.english
3. For hourOffset in 0..<24:
     entryDate = now + hourOffset hours
     totalHours = Int(entryDate.timeIntervalSince1970 / 3600)
     index = totalHours % pool.count
     entry = SimpleEntry(date: entryDate, text: pool[index], isRtl: isArabic)
4. Return Timeline(entries, policy: .atEnd)
   ← system requests new timeline after the last entry
```

Implications:
- **Deterministic across devices** — at 14:00 UTC on April 27 2026, every device shows phrase index `(timestamp / 3600) % count`.
- **No randomness** — phrase order rotates linearly through the array.
- The timeline is regenerated every 24 hours. WidgetKit may also evict the timeline earlier; `getTimeline` is best-effort scheduling.

### Phrase pools

- `SalahPhrases.arabic` — ~333 strings hardcoded in Swift. Largely matches `src/salahPhrases.ts` but they were edited independently and may drift.
- `SalahPhrases.english` — only **6** strings (line 341–348 of `WidgetExtension.swift`). When `user_lang == "en"`, the widget cycles through the same 6 strings every 6 hours.

This is a limitation worth knowing — the English experience on iOS is much weaker than the Arabic one.

### Visual design

- Background: deep navy gradient (`#07090f → #0d1220`) with a 2-color gold border stroke (`#e8c96a → #c9a84c`).
- Star ornament icon at top, gold `#c9a84c`.
- Phrase text: serif design, gold (`#e8c96a`), bold, size 14 (small) / 16 (medium), 4 lines max, scales down to 0.7×.
- RTL handled by `.environment(\.layoutDirection, ...)`.
- `Color(hex:)` extension at the bottom of the file.

### What the widget does NOT do

- ❌ Does not deep-link into the app on tap (no `widgetURL`)
- ❌ Does not respect `enable_active_timer` or `popup_speed` (those are Android-only concepts anyway)
- ❌ Does not use `salah_phrases` from Capacitor — its pool is hardcoded
- ❌ Does not animate (WidgetKit doesn't allow free-form animations)
- ❌ Does not support large or lock-screen widget sizes

## Known integration gap (important)

**The Capacitor Preferences plugin writes to `NSUserDefaults.standard`, but the widget reads from the App Group `UserDefaults(suiteName: "group.com.salo.alahmoha")`.** These are different stores. As a result, **changing the language inside the iOS app has no effect on the widget**.

Fix paths (pick one):
1. Configure the Capacitor Preferences plugin to use the App Group on iOS (set `Preferences.group` in `capacitor.config.json`). Verify the plugin version supports this.
2. Add a tiny Capacitor plugin that mirrors `user_lang` from `NSUserDefaults.standard` into the App Group on every change.
3. Add a manual sync step in `AppDelegate.applicationWillResignActive` or `applicationDidEnterBackground` that copies the value across.

See [08-known-issues.md](./08-known-issues.md) for full details.

## Build & run

- `npx cap open ios` opens the workspace in Xcode.
- Select the `App` scheme → run on simulator or device. The webview UI loads.
- To preview the widget in the simulator: long-press home screen → **+** → search "Salah Reminder" → add the small or medium variant.
- Widget timeline refreshes when iOS decides to (you can force in Xcode via Debug → Widget Kit Simulator, or kill & re-add the widget).

## What CapApp-SPM is

`ios/App/CapApp-SPM/` is the Swift Package Manager scaffolding Capacitor 7 uses to deliver the bridge as an SPM package instead of CocoaPods. You should not need to touch it.
