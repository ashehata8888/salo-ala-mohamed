# 01 — Overview

## What this app does

A reminder app that nudges the user to send Salawat (صلاة على النبي / "blessings upon the Prophet ﷺ") throughout the day. There is **no account, no backend, no online content** — every phrase is bundled in the binary.

## Identity

| Field | Value |

| Display name (Android) | `صلِّ على محمد ﷺ` (`strings.xml`) |

## Three user-facing surfaces

### 1. Settings UI (web → webview)
A single-screen React app rendered in the Capacitor webview. Lets the user:
- Toggle language (Arabic / English)
- Pick popup speed (slow / medium / fast — affects how long the overlay stays on screen)
- Toggle "hourly reminder" (on by default)
- Grant the Android `SYSTEM_ALERT_WINDOW` permission ("Draw over apps")

Settings are persisted to `@capacitor/preferences`. **The settings UI is the same on Android and iOS, but the overlay feature it controls only exists on Android.**

### 2. Android system overlay (native Java)
The actual feature. A gold-bordered Arabic/English Salawat phrase appears as a system overlay (drawn over other apps) at two triggers:
- **Every screen unlock** (with a 10-second cooldown to prevent spam)
- **Once per active hour** (timer pauses while screen is off, resets on every unlock)

The overlay auto-dismisses after a duration computed from text length and the user's "popup speed" setting. Tap or swipe sideways to dismiss early. A glowing animated border on the overlay acts as a visual countdown.

### 3. iOS home-screen widget (SwiftUI / WidgetKit)
A small/medium home-screen widget that displays a different Salawat phrase **every hour, on the hour**, in a premium dark-gold theme. The phrase index is deterministic: `Int(timeIntervalSince1970 / 3600) % phraseCount`, so the same phrase appears across all devices at the same wall-clock hour.

The widget is **not generated by Capacitor** — it is a separate Xcode target that must be added manually (see `iOS_SETUP_GUIDE.md` and [06-ios-widget.md](./06-ios-widget.md)).

## What it does NOT do

- ❌ No prayer-time / Adhan integration despite the project folder being `salahapp/`
- ❌ No notifications scheduled by date (only by unlock event + hourly timer when active)
- ❌ No syncing across devices
- ❌ No user accounts
- ❌ No analytics or telemetry
- ❌ No iOS overlay or unlock detection (iOS sandbox forbids this; iOS makes do with the widget)
- ❌ No Android home-screen widget
- ❌ No tests (no test runner configured)

## Source language

- **UI / phrases**: Arabic (default) and English
- **Code comments / native code**: English
- **Web app strings**: hardcoded in `src/i18n.ts` (no JSON files, no extraction)
